## Планы работы и отчёты агентов

**Назначение:** Долговременные планы работы агентов и отчёты о выполнении шагов по этим планам.

**Важно для LLM:**
- Папка `documentation/Plans/` **ВСЕГДА доступна для чтения и записи**.
- При начале новой сессии работы над задачей агент должен проверить наличие планов и отчётов в этой папке.

### Стандарты именования файлов

**Папка:** `documentation/Plans/`

**Формат файла плана:** `YYYY-MM-DD_feature-or-task-name_PLAN.md`
**Формат файла отчёта:** `YYYY-MM-DD_feature-or-task-name_PROGRESS.md`
**Важно:** Имя задачи (`feature-or-task-name`) должно совпадать в файле плана и соответствующем файле отчёта.

### Структура файла плана (СТАНДАРТ)

**Обязательные разделы:**

1.  **Заголовок с кратким описанием задачи** (включая Статус, Приоритет).
2.  **Контекст:** Краткое описание задачи, цели, и список **Затронутых файлов/модулей**.
3.  **Нумерованный список шагов:** Максимально атомарные и конкретные шаги.
4.  **Зависимости и примечания** (опционально).

### Структура файла отчёта (СТАНДАРТ)

**Обязательные разделы:**

1.  **Заголовок и ссылка на план** (включая Дату последнего обновления, Общий статус).
2.  **Текущий статус** (краткое резюме: выполнено X из Y шагов).
3.  **Детальный статус по шагам:** Для каждого шага указать **Статус** (DONE/IN PROGRESS/TODO), **Изменения**, и **Затронутые файлы**.
4.  **Примечания и проблемы** (опционально).
